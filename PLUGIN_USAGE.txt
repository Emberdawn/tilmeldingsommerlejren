Sommerlejr Tilmelding WordPress Plugin - Usage Guide

Overview
Sommerlejr Tilmelding is a WordPress plugin for summer camp registration.
It lets logged-in users save a draft registration, upload proof of payment,
submit the registration for approval, and receive status emails. Site
administrators can configure prices, edit email templates, review pending
registrations, approve registrations, move approved registrations back to
pending, and delete registrations.

Installation
1. Copy summer-camp-registration.php into wp-content/plugins/.
   You may place it directly in that folder or inside its own plugin folder.
2. In WordPress admin, go to Plugins.
3. Activate "Sommerlejr Tilmelding".
4. After activation, the plugin creates its registration database table and
   grants the required capabilities to the administrator role.

Main Registration Page
Create or edit a WordPress page and insert this shortcode:

[summer_camp_registration]

Only logged-in users can use the registration form. Visitors who are not
logged in will see a message telling them to log in.

On the form, users can enter:
- Number of adults
- Number of children
- Number of day tickets
- Payment transfer screenshot or receipt, as an image or PDF

The plugin calculates the total price automatically from the configured
prices. Users can save their registration as a draft. The "Send til
godkendelse" button becomes available when the registration has at least one
participant or day ticket and an uploaded payment screenshot or receipt.

Registration Status Flow
The plugin uses these statuses:

draft:
The user has saved information but has not submitted it for approval.

submitted:
The user has sent the registration for approval. The user can use "Lav
rettelser" to move it back to draft and make changes.

approved:
An administrator has approved the registration. The form becomes locked for
that user and shows the approved registration summary.

Admin Menu
After activation, WordPress admin gets a "Sommerlejr" menu.

Pris indstillinger:
Set the prices used by the form:
- Adult full camp price
- Child full camp price
- Day ticket price

Mail indstillinger:
Set the sender name, sender email, submitted email, and approved email.
The email templates can use these placeholders:
- {display_name}
- {email}
- {adults}
- {children}
- {day_tickets}
- {total_price}

Alle tilmeldinger:
Shows all registrations, including their user, email, participant counts,
price, status, and uploaded proof. Registrations can be deleted here.

Ikke godkendte:
Shows submitted registrations waiting for approval. Administrators can open
the uploaded proof and approve registrations from this page.

Additional Shortcodes
The plugin also provides these shortcodes:

[summer_camp_registration_stats]
Shows a small public status widget with total adults, children, and day
tickets from submitted and approved registrations.

[summer_camp_pending_registrations]
Shows pending registrations on a front-end page. The viewer must be logged in
and have permission to view pending registrations.

[summer_camp_approved_registrations]
Shows approved registrations on a front-end page. The viewer must be logged in
and have permission to view approved registrations. Approved registrations can
be moved back to pending from this table when the viewer has permission.

Permissions
On activation, the plugin grants these capabilities to administrators:
- view_summer_camp_pending
- view_summer_camp_approved
- approve_summer_camp_registrations
- edit_summer_camp_registration_status
- delete_summer_camp_registrations

Administrators also retain access through manage_options fallback checks in
several plugin screens.

Uploaded Payment Proof
Users upload payment proof through the registration form. Image uploads show a
thumbnail preview, and PDF uploads are linked as files. In the admin pending
table, image proof can be opened in a larger view with zoom controls.

Email Notifications
The plugin sends an email when a registration is submitted and another email
when it is approved. Configure the content under Sommerlejr -> Mail
indstillinger. WordPress mail delivery must be working on the site for these
messages to arrive.

Data Storage
The plugin stores registration data in a custom database table named:

wp_summer_camp_registrations

The actual prefix may differ if the WordPress installation uses a custom
database prefix. Each user has one registration row, so saving again updates
that user's existing registration.

Typical Setup Checklist
1. Install and activate the plugin.
2. Add [summer_camp_registration] to the registration page.
3. Configure prices under Sommerlejr -> Pris indstillinger.
4. Configure email sender and templates under Sommerlejr -> Mail indstillinger.
5. Ask users to log in before registering.
6. Review submitted registrations under Sommerlejr -> Ikke godkendte.
7. Approve valid registrations after checking the uploaded payment proof.
